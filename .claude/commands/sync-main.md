mainブランチに切り替えて最新化し、リモートで削除済みのローカルブランチをクリーンアップする。「mainに戻して」「ブランチを整理」「ブランチを最新に」「不要なブランチを削除」「sync」など、ブランチの整理・同期に関するリクエストで使用する。

## 手順

### 1. mainブランチへの切り替えと最新化

```bash
git checkout main && git fetch --all --prune && git pull
```

`git fetch --prune` により、リモートで既に削除されたリモート追跡ブランチがローカルからも除去される。

### 2. 削除対象ブランチの検出

```bash
git branch -vv | grep ': gone]'
```

リモート追跡ブランチが `gone`（リモートで削除済み）になっているローカルブランチを検出する。

該当ブランチがなければ「削除対象のブランチはありません。」と表示してステップ4へスキップする。

### 3. 削除の確認と実行

AskUserQuestionツールで確認する：
- 質問文: 「以下のローカルブランチはリモートで削除済みです:\n（該当ブランチ名を列挙）\n削除しますか？」
- 選択肢: ["全て削除する", "中止"]
- 「中止」→ ブランチ削除をスキップしてステップ4へ

確認後、各ブランチを削除する：
```bash
git branch -d <branch-name>
```

`-d`（安全削除）を使う。マージされていないブランチがあれば警告が出るので、その場合はユーザーに `-D`（強制削除）するか確認する。

### 4. 結果の表示

実行結果のサマリーを表示する：
- 現在のブランチ: main（最新）
- 削除したブランチ数と名前（あれば）
- マージ未完了で削除できなかったブランチ（あれば）
