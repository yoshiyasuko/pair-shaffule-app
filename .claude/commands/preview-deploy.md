GASへのプレビューデプロイを作成する。productionデプロイとは別のURLで動作確認できるよう、`[PREVIEW]` プレフィックス付きの新規デプロイを作成・管理する。productionのURLやデプロイには一切触れない。

## 手順

### 1. 現在の状態を把握

以下を並列で実行：
- `git status` — 未コミットの変更を確認
- `clasp deployments` — 既存デプロイメント一覧を取得

### 2. 未コミット変更の処理

未コミットの変更がある場合、AskUserQuestionツールで確認する：
- 質問文: 「未コミットの変更があります。コミットしてからプレビューデプロイしますか？」
- 選択肢: ["コミットしてデプロイ", "コミットせずデプロイ", "デプロイを中止"]
- 「コミットしてデプロイ」→ Skillツールで `commit` スキルを実行する（引数: "skip-push skip-hooks"）
- 「コミットせずデプロイ」→ そのまま続行（WIP確認に便利）
- 「デプロイを中止」→ 処理を終了

### 3. 既存プレビューデプロイの確認

手順1のデプロイメント一覧から `[PREVIEW]` を含むものを探す。

- **ある場合** — AskUserQuestionツールで確認する：
  - 質問文: 「既存のプレビューデプロイが見つかりました:\n（該当デプロイのID・メッセージを列挙）\n既存のプレビューデプロイを上書きしますか？」
  - 選択肢: ["上書きする", "新規作成する", "デプロイを中止"]
  - 「上書きする」→ 既存IDを使って更新（URL維持）
  - 「新規作成する」→ 新規デプロイを作成
  - 「デプロイを中止」→ 処理を終了
- **ない場合** → 新規デプロイを作成

### 4. clasp push

```bash
clasp push
```
失敗した場合はエラーを表示して処理を終了する。

### 5. デプロイメッセージの生成

`[PREVIEW]` プレフィックス付きのメッセージを生成する。ブランチ名と変更内容を含めることで、どの作業のプレビューか後から識別しやすくする。

フォーマット: `[PREVIEW] <ブランチ名>: <変更内容の要約>`
例: `[PREVIEW] feature/new-ui: Add card flip animation`

### 6. デプロイの実行

手順3の結果に応じて実行：
```bash
# 上書きの場合
clasp deploy -i <deploymentId> -d "<デプロイメッセージ>"
# 新規作成の場合
clasp deploy -d "<デプロイメッセージ>"
```

### 7. 結果の表示

デプロイ完了後に以下を表示する：
- デプロイメントID
- デプロイメッセージ
- WebアプリURL: `https://script.google.com/macros/s/<deploymentId>/exec`
- 不要になったら `/kill-all-preview` で削除できること
